## 三、需要你提供/确认的输入

为进入编码阶段顺畅，以下信息越早提供越好：

1. **LLM API Key 与可用模型清单**（决定 T2 路由表具体填什么）。
2. **目标学段优先级**（K12？高校？成人？影响 Prompt 与合规强度）。
3. **首批专业模板的风格方向**（学术/活泼/极简…，影响 E1 设计）。
4. **部署目标环境**（云厂商/区域，影响合规与对象存储选型）。
5. **产品正式名**（暂定「言课」，影响品牌化文案与 manifest）。

## 四、我的建议（若你希望我直接定夺）

- **P1**：采纳 PRD 建议——MVP 不做实时互动，仅在数据模型预留互动块。这是范围纪律的关键。
- **P2**：MVP 先支持「展示设备直接打开链接全屏」这条最稳路径；「手机遥控+大屏」待 Phase 2 实地验证后定。
- **P3**：MVP 不涉及学生端，问题在 Phase 2 前再定；倾向「课堂码 + 匿名/轻身份」以降低未成年人合规负担。
