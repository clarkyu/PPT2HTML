# 05 · MVP 范围与功能映射

> MVP 聚焦验证三件核心事（PRD 第七章）：
> 1. 「一句话／导入 PPT → 大纲确认 → 精修」能否稳定产出令老师满意、风格可控的课件；
> 2. 老师是否愿意并能够「以网页版课件在浏览器中直接授课」；
> 3. 存量 PPT 用户能否被「导入改造」有效承接。

## MVP 必达验收标准（Definition of Done）

围绕三件核心事，逐条可验证：

**核心事 ①：稳定产出满意、风格可控的课件**
- [ ] 输入一句话（含可选约束），≤ 秒级返回意图卡片，要素可一键修改。
- [ ] 确认意图后 ≤ 秒级返回大纲，可增删改调序。
- [ ] 确认大纲后 1–2 分钟内渐进式产出完整课件，学科准确、学段适配、含教学法结构。
- [ ] 可一键切换至少 3 套专业模板，内容不变、风格立变。
- [ ] 可基础自定义配色/字体/版式/校徽。
- [ ] 自然语言/直接编辑可对单块做局部修改，不洗牌全篇。

**核心事 ②：网页版课件浏览器直接授课**
- [ ] 课件有稳定链接，打开即可全屏播放。
- [ ] 手机/平板触控翻页；桌面/大屏键鼠/遥控翻页。
- [ ] 演讲者备注与计时随端呈现。
- [ ] 各端布局自适应、可读。
- [ ] 已生成课件可离线打开。

**核心事 ③：存量 PPT 导入承接**
- [ ] 上传 .pptx，抽取文本/图片/结构成功率达标（常见版式）。
- [ ] 映射为课件并套模板，产物与一句话生成同构、可编辑可播放。
- [ ] 异常/复杂文件有清晰失败提示与降级。

**基础设施**
- [ ] 注册登录可用；课件云端保存、命名、检索、再编辑。
- [ ] PDF 导出可用。
- [ ] 可添加到主屏。

## 明确不做（MVP 边界）

- 实时课堂互动 live 会话、学生端扫码加入、师生/生生实时互动、学情回流（**Phase 2 头号目标**）
- 手机遥控 + 大屏配对式演讲者模式
- 语音输入、上传大纲、多模态输入
- PPT 深度 AI 改造（重组/增强/建议互动）
- 自建模板、另存复用、模板市场
- 多版本对比生成、风格记忆个性化
- .pptx 导出、长图导出
- 学生身份体系、组织/班级管理、协作编辑、权限体系

> 互动虽不做实时，但 **数据模型从第一天就把互动块作为一等公民**（见 03-data-model），
> 确保 Phase 2 接入实时层时无需返工。这是 MVP 与 Phase 2 之间最关键的架构衔接。
