# 06 · 分阶段路线图

> 对应 PRD 第十二章三阶段，落为可执行的里程碑。本节给出顺序与依赖，不绑定具体日期
> （待团队与资源确定后排期）。

## 阶段总览

```
Phase 1 (MVP)            Phase 2 (互动·头号目标)        Phase 3 (平台化)
生成闭环+直接授课+存量承接  实时课堂+学生端                 统一身份+规模化
─────────────────────►   ─────────────────────►        ─────────────────────►
```

## Phase 1 · MVP（验证生成闭环、网页版直接授课、存量承接）

按依赖关系分为 6 个里程碑，建议顺序推进，M0/M1 是地基。

### M0 · 地基（数据模型 + 脚手架）
- 课件 Schema（TS 类型 + Zod 校验）—— 一切的枢纽，最先冻结
- Next.js + Tailwind + Prisma + Auth 脚手架
- LLM Provider 抽象层骨架 + 一个可跑通的适配
- 渲染层骨架（能把一份手写 Deck JSON 渲染出来）
> **出口标准**：手写一份 Deck JSON，能在浏览器中渲染并翻页。

### M1 · 渲染与播放（先有「呈现」再有「生成」）
- 块渲染器（全部 MVP 块类型）
- 多端自适应布局引擎
- 全屏播放器：翻页、演讲者备注、计时（F9）
- 课件链接路由 `/deck/[id]` 与 `/deck/[id]/play`（F7）
> **出口标准**：任意 Deck 都能在手机/桌面正确呈现并全屏授课。

### M2 · 生成闭环（产品的核心赌注）
- 意图解析 Agent + 要素卡片 UI（F1、F2）
- 教学设计 Agent + 大纲编辑 UI（F3）
- 内容生成 Agent + 流式渲染（F4）
- 校验 Agent
> **出口标准**：一句话 → 意图确认 → 大纲确认 → 完整课件渐进呈现。

### M3 · 模板与精修
- 模板系统（≥3 套专业模板）+ 一键切换实时预览（F5）
- 基础自定义：配色/字体/版式/校徽（F5）
- 对话式精修 + 直接编辑（块级，F8）
> **出口标准**：换模板内容不变；可局部精修不洗牌全篇。

### M4 · 存量承接（PPT 导入）
- .pptx 上传与 OOXML 解析（文本/图片/结构）
- 映射到课件 Schema + 套模板（F6 基础迁移）
- 失败兜底与降级提示
> **出口标准**：常见 .pptx 导入后可编辑、可播放，与一句话产物同构。

### M5 · 收口（账户、存储、PWA、导出）
- 注册登录 + 个人作品空间（F15）
- 课件云端保存、命名、检索、再编辑（F14）
- PWA：添加主屏、离线打开、授课预加载（F13）
- PDF 导出（F14）
> **出口标准**：MVP 验收清单全绿（见 05-mvp-scope），可交付小批真实老师试用。

### MVP 关键依赖链

```
M0 数据模型 ──► M1 渲染播放 ──► M2 生成闭环 ──► M3 模板精修 ──┐
       └────────────────────► M4 PPT 导入 ──────────────────┤──► M5 收口
```
M4（PPT 导入）只依赖 M0 的数据模型与 M1 的渲染，可与 M2/M3 **并行**。

## Phase 2 · 课堂互动与学生端（头号目标）

- 实时基础设施：WebSocket 服务 + Redis Pub/Sub + 课堂会话管理
- 学生端：扫码/课堂码加入（匿名或轻身份）
- 师生互动：推送投票/选择/判断/限时测验、实时结果聚合、随机点名/抢答、「不懂」反馈、推送当前页（F10）
- 生生互动：讨论墙（弹幕）、分组协作、同伴互评/投票选优（F11）
- 学情数据回流：作答率/正确率/参与度实时 + 课后回顾（F12）
- 同期：手机遥控 + 大屏配对演讲者模式；PPT 深度 AI 改造
> 互动块的数据模型已在 MVP 就绪，本阶段为其注入运行时能力。

## Phase 3 · 平台化与规模化

- 接入统一身份体系（手机号 IdP）
- 组织/班级管理、协作编辑、权限体系
- 自建模板、另存复用、模板市场
- 深度学情分析与个性化
- .pptx 导出等兼容兜底

## 阶段间的架构连续性

| 衔接点 | MVP 已做的准备 |
| --- | --- |
| MVP → P2 互动 | 互动块是数据模型一等公民，`runtime.live` 开关即接入实时 |
| MVP → P2 学生端 | 课件即链接的寻址能力，学生加入复用同一寻址 |
| MVP → P3 身份 | Auth 以手机号为主锚点，预留 IdP 对接接口 |
| MVP → P3 模板市场 | 模板注册表 + 内容/渲染分离，模板可独立分发 |
